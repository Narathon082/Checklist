# 📊 ระบบตรวจประเมินคุณภาพข้อมูล (DQA Checklist)
> **สำนักงานปลัดกระทรวงพาณิชย์ (Office of the Permanent Secretary - OPS)**

ระบบเว็บแอปพลิเคชันเพื่ออำนวยความสะดวกในการกรอกแบบตรวจประเมินคุณภาพข้อมูล (Data Quality Assessment - DQA Checklist) ด้วยตนเอง พัฒนาโดยใช้โครงสร้าง PHP, HTML, CSS, JavaScript และฐานข้อมูล MySQL

เพื่อตอบสนองความต้องการของผู้ใช้งานที่ต้องการให้รวมแบบฟอร์มประเมินทั้งหมดเข้ามารวมเป็น **หน้าจอเดียวแบบ Multi-step** โดยยังคงการนำเสนอด้วยปุ่มเปลี่ยนสเต็ปย่อยและโครงสร้างเนื้อหาข้อความดั้งเดิมครบถ้วน 100% ไม่สูญหาย ระบบจึงได้พัฒนาไฟล์ใหม่ชื่อ [dqa_combined.php](file:///c:/xampp/htdocs/Checklist/dqa_combined.php) เพื่อทำงานร่วมกับส่วนอื่นของระบบได้อย่างสมบูรณ์

## ⚡ คุณสมบัติเด่นของหน้าประเมินรวม (dqa_combined.php Key Features)

หน้าประเมินรวมแบบเบ็ดเสร็จในหน้าเดียว [dqa_combined.php](file:///c:/xampp/htdocs/Checklist/dqa_combined.php) มาพร้อมระบบ JavaScript Engine ทรงประสิทธิภาพที่ควบคุมลักษณะการใช้งานขององค์กรระดับพรีเมียมดังนี้:

1. **ระบบ Stepper & Multi-step Form**:
   * รวมเนื้อหาจากแบบฟอร์ม 4 หน้าหลักมาซ้อนกันแบบสลับแสดงผลด้วย Class `.form-step.active`
   * มีตัวแสดงผลความคืบหน้า (Dynamic Stepper Line Progress) ด้านบน ช่วยชี้แจงหัวข้อหลักที่กำลังทำงาน (ข้อมูลทั่วไป -> มิติคุณภาพข้อมูล -> ประเมินตนเอง -> การควบคุมติดตาม)
   * รองรับการคลิกย้อนกลับ (Prev) หรือถัดไป (Next) และสามารถกดเลือกหัวข้อที่ Stepper ด้านบนโดยตรงเพื่อสลับหน้าได้ (เฉพาะหน้าที่เคยผ่านการตรวจสอบการกรอกแล้ว)

2. **ระบบตรวจสอบฟิลด์ข้อมูลแบบละเอียด (Step-by-Step Validation)**:
   * ในทุกๆ ครั้งที่กดย้ายสเต็ปถัดไป ฟังก์ชัน `validateStep()` จะตรวจสอบค่าอินพุตบังคับและคะแนนวิทยุ (Radio) ในขั้นตอนนี้ทั้งหมด
   * หากมีฟิลด์ว่าง ระบบจะไฮไลต์ช่องกรอก หรือขอบเขตแถวตารางเกณฑ์/การ์ดคำถามเป็น **สีแดง** ทันทีเพื่อความเด่นชัด
   * ระบบจะแสดง **Custom Alert Modal Box** ที่แสดงรายการฟิลด์หรือคำถามที่ผู้ใช้ยังกรอกไม่ครบถ้วนทั้งหมด และจะทำการเลื่อนหน้าจอ (Smooth Scroll Focus) ไปที่อินพุตจุดแรกที่ยังไม่ได้กรอกโดยอัตโนมัติ

3. **การซิงโครไนซ์ข้อมูลข้ามสเต็ป (Cross-Step Data Syncing)**:
   * เมื่อผู้ใช้พิมพ์กรอก "ชื่อข้อมูล" (info_title) และ "ชื่อหน่วยงาน" (info_agency) ในสเต็ปที่ 1 ข้อมูลดังกล่าวจะแสดงผลแบบประสานเวลารวม (Real-time Syncing) ไปโชว์เป็นกล่องแสดงข้อมูลแบบอ่านอย่างเดียว (Read-only) ด้านบนของสเต็ปที่ 2, 3 และ 4 ทันที ป้องกันความเข้าใจผิดในระหว่างประเมิน โดยไม่มีการใช้ชื่อฟิลด์ Input ซ้ำกันในโค้ดฟอร์ม

4. **การคำนวณและวาดกราฟใยแมงมุมเรดาร์ (Real-time Radar Chart & Score Table)**:
   * ในสเต็ปที่ 3 (แบบประเมินตนเอง) ทุกการคลิกเลือกคะแนนประเมิน (sa_1_1 ถึง sa_5_5) ระบบจะคำนวณหาค่าเฉลี่ยคะแนนของแต่ละมิติ (จาก 5 มิติคุณภาพ) ในทันที
   * แสดงผลคะแนนในรูปแบบกราฟใยแมงมุมสถิติ (Chart.js Radar Chart) แบบเคลื่อนไหวเรียลไทม์
   * อัปเดตตารางค่าเฉลี่ยรายมิติ และคำนวณคะแนนรวมเฉลี่ยสุทธิ (Overall Average Score) สูงสุด 4.00 คะแนน พร้อมกำหนดช่วงสีอัตโนมัติ (เขียว-ดีมาก, น้ำเงิน-ดี, เหลือง-ปานกลาง, แดง-ต่ำ)

5. **ระบบ autosave อัจฉริยะ (Debounced Autosave & Local Backup)**:
   * ทุกการพิมพ์ ข้อความ หรือการคลิกอินพุต ระบบจะบันทึกแบบร่างโดยอัตโนมัติแบบชะลอเวลา (Debounced 1.5 วินาที) ไปยัง API เพื่ออัปเดตลงฐานข้อมูล MySQL
   * เมื่อกดคลิก "ถัดไป" หรือ "ย้อนกลับ" ระบบจะเซฟข้ามหน้าทันที
   * หากระบบตรวจพบปัญหาด้านการเชื่อมต่อเครือข่าย ระบบจะ autosave แบบร่างสำรองเก็บไว้ที่ `localStorage` ของเบราว์เซอร์แทน และแจ้งเตือนผู้ใช้งานทราบ เมื่อกลับมาเปิดใหม่จะโหลดข้อมูลร่างอัตโนมัติทันที

6. **ระบบล้างข้อมูลและการพิมพ์รายงานมาตรฐาน (Reset & Print-friendly)**:
   * **ปุ่มล้างข้อมูล (Reset):** ส่งคำขอรีเซ็ตข้อมูลผ่าน API ไปลบประวัติในฐานข้อมูล ล้างใน Local Draft และรีโหลดหน้าใหม่เพื่อให้สามารถเริ่มประเมินใหม่ได้สะอาด
   * **ปุ่มพิมพ์รายงาน (Print):** ตรวจสอบการกรอกข้อมูลครบถ้วนทุกหน้า หากครบจะนำทางไปเปิดสั่งพิมพ์มาตรฐานของเบราว์เซอร์ (Print Window) ทันที โดยมี CSS `@media print` ซ่อนปุ่ม ปุ่มเปลี่ยนหน้า และองค์ประกอบตกแต่งเว็บ เพื่อความสวยงามเหมือนใบตรวจเอกสารราชการทั่วไป

## 📁 โครงสร้างไฟล์ในระบบ (File Categorization)

### 1. ⚙️ ไฟล์โครงสร้างหลักและการเชื่อมต่อฐานข้อมูล (Core & Database)
*   [db.php](file:///c:/xampp/htdocs/Checklist/db.php): จัดการการเชื่อมต่อฐานข้อมูล MySQL มีระบบ **Auto-Migration** ที่จะตรวจจับหากไม่มีตาราง `submissions` และ `agencies` และทำการสร้างตารางพร้อมติดตั้งข้อมูลรายชื่อหน่วยงาน 12 แห่งให้โดยอัตโนมัติในการรันครั้งแรก
*   [database.sql](file:///c:/xampp/htdocs/Checklist/database/database.sql): ไฟล์ DDL/DML สำหรับนำเข้าฐานข้อมูลเพื่อความรวดเร็วผ่าน phpMyAdmin

### 2. 🔌 ส่วนประมวลผลข้อมูล (API & Session Handling)
*   [api.php](file:///c:/xampp/htdocs/Checklist/api.php): API Endpoint คอยรับและส่งต่อข้อมูลในฟอร์มแบบประเมินในรูปแบบ JSON และบันทึก/อัปเดตข้อมูลลงฐานข้อมูล MySQL ตามคอลัมน์ที่รองรับ (มีฟังก์ชันเช็กคอลัมน์อัตโนมัติ)
*   [new.php](file:///c:/xampp/htdocs/Checklist/new.php): ทำความสะอาด Session การประเมินเดิม แล้วส่งไปยังหน้ากรอกประเมินต้นฉบับ
*   [edit.php](file:///c:/xampp/htdocs/Checklist/edit.php): รับ ID แบบประเมินเพื่อตั้งค่าลงใน Session สำหรับเปิดขึ้นมาแก้ไขข้อมูล

### 3. 📝 หน้าฟอร์มแบบประเมินดั้งเดิมแยกหน้า (Multi-page Forms)

> ไฟล์เหล่านี้ถูกเก็บรักษาไว้ตามโครงสร้างเดิมโดยไม่มีการเข้าไปแก้ไขทับแต่อย่างใด เพื่อให้คงระบบการกรอกดั้งเดิมของหน่วยงานไว้

*   [index.php](file:///c:/xampp/htdocs/Checklist/index.php): หน้าแรกสำหรับกรอกข้อมูลทั่วไปและเกณฑ์ประเมินเบื้องต้น
*   [page3.php](file:///c:/xampp/htdocs/Checklist/page3.php): หน้าประเมินมิติคุณภาพข้อมูล
*   [page4.php](file:///c:/xampp/htdocs/Checklist/page4.php): หน้าแบบประเมินตนเองพร้อมระบบกราฟเรดาร์
*   [page5.php](file:///c:/xampp/htdocs/Checklist/page5.php): หน้าประเมินเกณฑ์การควบคุมและติดตามคุณภาพข้อมูล

### 4. 🎛️ หน้าประเมินรวมสเต็ปเดียว (Combined Form)
*   [dqa_combined.php](file:///c:/xampp/htdocs/Checklist/dqa_combined.php): **หน้าที่รวมหน้าฟอร์มทั้งหมดไว้เป็นหน้าเดียว** คัดลอกข้อมูลและเกณฑ์ของหน้าดั้งเดิมมาครบถ้วน พร้อมระบบนำทาง Stepper และ Javascript Engine ควบคุมการประเมินเบ็ดเสร็จ

### 5. 🖥️ หน้าแสดงผลและภาพรวม (Views & Dashboard)
*   [dashboard.php](file:///c:/xampp/htdocs/Checklist/dashboard.php): หน้ารายการประเมินคุณภาพทั้งหมด แสดงสถิติจำนวนฟอร์มที่กรอกเสร็จและแบบร่าง ค้นหาตามชื่อชุดข้อมูล กรองข้อมูลตามหน่วยงาน และลบข้อมูลได้
*   [view.php](file:///c:/xampp/htdocs/Checklist/view.php): หน้าดูรายงานผลแบบประเมินคุณภาพฉบับเต็มอย่างเป็นทางการ เพื่อความสะดวกในการพิมพ์และเก็บหลักฐานอ้างอิง

### 6. 🎨 สไตล์ชีตและรูปภาพ (Assets & Styles)
*   [style.css](file:///c:/xampp/htdocs/Checklist/assets/css/style.css): ควบคุมสไตล์และธีมความสวยงามของทุกหน้าเว็บอิงตามอัตลักษณ์สีน้ำเงิน-ทอง (MOC Blue & Gold)
*   [app.js](file:///c:/xampp/htdocs/Checklist/assets/js/app.js): คุมการทำงานฟังก์ชั่น autosave ในเวอร์ชันฟอร์มแบบแยกหน้าเดิม
*   [ops-logo.jpg](file:///c:/xampp/htdocs/Checklist/assets/images/ops-logo.jpg): โลโก้ของสำนักงานปลัดกระทรวงพาณิชย์

## 🛠️ ขั้นตอนการติดตั้งและเปิดใช้งาน (Installation Guide)

### 1. เปิดจำลองเซิร์ฟเวอร์ด้วย XAMPP
*   ดาวน์โหลดและติดตั้ง **XAMPP**
*   เปิด **XAMPP Control Panel** แล้วกดปุ่ม **Start** ที่แถว **Apache** และ **MySQL**

### 2. นำไฟล์โครงการไปวาง
*   นำโฟลเดอร์โครงการนี้ไปวางไว้ที่ตำแหน่ง:
    `C:\xampp\htdocs\Checklist`

### 3. เปิดเรียกใช้งานและตั้งค่าฐานข้อมูล
*   ระบบมีฟีเจอร์ **Auto-Migration** คุณสามารถเปิดเรียกหน้าเว็บผ่านทางเว็บเบราว์เซอร์ได้ทันทีที่:
    [http://localhost/Checklist/dashboard.php](http://localhost/Checklist/dashboard.php)
*   ระบบจะเชื่อมต่อ MySQL และสร้างฐานข้อมูลชื่อ `checklist` รวมถึงสร้างตาราง `submissions` และ `agencies` พร้อมนำเข้าข้อมูลรายชื่อหน่วยงานให้เองทันทีโดยที่ผู้ใช้ไม่ต้องเข้าไปกดนำเข้า SQL ใดๆ เพิ่มเติม

> [!TIP]
> หากต้องการนำเข้าฐานข้อมูลด้วยตนเอง สามารถนำเข้าไฟล์ [database.sql](file:///c:/xampp/htdocs/Checklist/database/database.sql) ผ่านเว็บ `http://localhost/phpmyadmin/` ได้โดยตรงเช่นเดียวกัน
